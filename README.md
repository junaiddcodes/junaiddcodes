# Hi, I'm Junaid Asif

I'm a full stack software engineer with around four years of experience building and maintaining SaaS applications. Most of my work has been with React, Next.js, Node.js, NestJS, and TypeScript, although I've also worked on projects using Angular and Vue when the product required it.

For the last four years I've worked entirely in remote teams, building products used across healthcare, agriculture, enterprise software, live commerce, document automation, and digital media. I enjoy working on products that already have users and solving the kinds of problems that come with growing software, whether that's refactoring legacy code, improving performance, designing new features, or making systems easier to maintain.

A lot of my experience comes from stepping into existing codebases rather than starting from scratch. I like understanding how a system works before changing it, and I'm usually the happiest when I'm simplifying architecture, removing technical debt, or tracking down the root cause of an issue that's been around for a while.

## How I like to work

I don't mind joining a project halfway through its lifecycle. In fact, that's where I've spent most of my career.

Most of the products I've worked on were already in production, so a big part of my job has been understanding existing architecture, shipping new features without breaking what's already there, and gradually improving the codebase over time. That has included frontend migrations, API development, payment integrations, performance tuning, debugging production issues, and modernizing older parts of applications while keeping releases stable.

I like straightforward code, predictable architecture, and systems that are easy for the next developer to understand.

## AI in my workflow

I use AI every day, but mostly as a productivity tool rather than something that writes applications for me.

Cursor is my primary editor, and I regularly use Claude Code to think through implementation approaches, review architecture decisions, and speed up repetitive tasks. I also use Lovable when prototyping interfaces and n8n for automating internal workflows and integrations.

These tools help me move faster, but they're only part of the workflow. The engineering decisions, architecture, debugging, and production ownership are still mine.
